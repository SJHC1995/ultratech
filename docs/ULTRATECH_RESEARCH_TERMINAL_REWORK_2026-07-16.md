# 研究终端界面重构记录

发布日期：2026-07-16

## 本次改造

研究终端改为专用研究界面，不再承载玩家物品栏、身份卡槽或任何物品输入输出。终端只读取研究进度、网络 FLOPS 产出和当前研究请求状态。

- 使用终端原始底图作为整体布局，左侧为可滚动研究列表，右侧为研究说明和操作图标。
- 研究列表接入 92x22 蜂窝底纹切片，避免使用普通纯色卡片。
- 黄色表示可研究，淡蓝色表示预选/悬停，深蓝色表示已选，绿色表示正在研究或已解锁，红色表示研究不可用。
- 已开始研究的条目显示实际研究进度；按住 ALT 可查看历史档案。
- 右侧详情按可研究、研究中、已解锁、不可用四种状态切换描述、解锁结果与操作图标。

## 解码监视器

原玩家物品栏区域替换为黑色解码监视器。

- 左侧保留终端在线状态、当前研究最低 FLOPS、网络 FLOPS 产出与在线设备数。
- 只有在研究已经开始，且当前网络 FLOPS 产出达到该研究最低 FLOPS 时，右侧才会滚动绿色解码字符与扫描线。
- 研究已开始但 FLOPS 不足时，监视器显示 `ERROR：算力不足`，不播放解码字符。
- 待机时显示等待研究状态。

## 本地化与资源加载

研究终端新增的标题、状态、错误提示和描述全部由语言键提供。构建流程现在会在 `processResources` 阶段将 DataGen 生成的语言键合并进最终资源包，避免客户端显示 `gui.ultratech...` 这类未解析键。

- Gradle `build`
