# UltraTech 开发日志 - 2026-07-19：算力请求身份化与广播仲裁

## 为什么要改

此前所有机器的缺算力需求最终会汇入一个匿名总量。处理器面板因此可能在任务结束后仍显示旧的需求值；当总量超过单台推理算力终端的冗余额度时，也无法指出是哪一台机器提出了超额申请。

本次改造将算力网络从“总量广播”变成“带身份的请求广播”。游戏内显示仍使用 `FloPS`，不会新增玩家需要理解的内部单位。

## 请求契约

每一条研究或机器算力请求都有一个八位十六进制 UUID。请求记录包含来源、请求量、UUID 与状态；同一来源持续运行时保留同一个 UUID，直到广播接受、请求完成、来源清除或玩家明确重新发起。

| 状态 | 广播行为 | 机器行为 |
|---|---|---|
| 等待处理器 | 暂不广播，不拉黑 | 安装或启用处理器后继续等待 |
| 可承载 | 进入带 20% 冗余的广播额度 | 写入玩家网络缓存 |
| 超出已安装处理器额度 | 只拉黑当前 UUID | 该 UUID 不再被广播 |
| 重新发起 | 创建新的 UUID | 新请求重新参与仲裁 |
| 无活跃请求 | 需求与广播值立即归零 | 不显示历史残留 |

黑名单针对 UUID，而不是研究 ID、机器类型或玩家。升级处理器不会让被拒绝的旧 UUID 自动复活；研究终端重新点击开始会创建新 UUID。对有明确操作入口的航天、探测、部署和蓝图任务，重新执行相应操作同样会建立新的请求。

## 已接入系统

- 研究终端与蓝图研究终端。
- 推理算力终端与算力广播终端。
- 无线能量请求端。
- 航天器导航台、探测器发射台、中继信号站发射台。
- 探测收发终端与探测监视终端。
- 空间工业终端、部署计划与采矿机器人网络。
- 科幻巨构控制器。

放置式机器的来源键由机器类型、维度与方块坐标组成，因此不同维度或不同位置的同类机器不会共享请求 UUID。

## 存档迁移

`PlayerResearchSavedData` 升级到数据版本 `6`：

- 版本 `5` 的研究 UUID、来源与黑名单继续读取。
- 版本 `6` 新增机器请求表和机器请求 UUID 黑名单。
- 版本 `5` 及更早版本的匿名聚合请求会在首次读取时清除。它们没有来源或 UUID，保留只会造成“幽灵需求”。
- 已有研究会在下一次正常运行时按当前研究状态继续建立有效请求，不会因为迁移损失研究进度。

## 面板修正

推理算力终端的“需求”读数只表示当前可追溯请求。若没有研究请求、机器请求或遗留队列，`需求` 与 `请求` 均显示 `0 FloPS`，不再保留上一轮的历史数值。
