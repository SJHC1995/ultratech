# UltraTech 整体规划与当前路线图

更新日期：2026-07-02  
项目：UltraTech  
Mod ID：`ultratech`  
包路径：`com.halfsword.ultratech`  
目标环境：Minecraft Forge 1.20.1 / Forge 47.4.4 / Java 17

## 1. 项目定位

UltraTech 是一条从畜力、风水机械、蒸汽、电力、核工业、聚变、高能物理一路推进到星际工业与终局科技的大型工业模组路线。核心目标不是堆更多机器方块，而是让玩家逐级建立真实的能源、物流、研究、算力、材料与星球资源体系。

当前设计原则：

- 早期机器使用 SU（Strain Unit）作为实时机械能，低阶段机器不在内部缓存 SU。
- 中后期逐步引入 FE、电力网络、热力、气体、高级能量、算力与研究门槛。
- 研究进度绑定玩家知识库，不绑定单个机器方块。
- 重要机器必须拥有明确输入、输出、维护、环境影响、GUI、Tooltip、JEI/TOP 信息和 DataGen 资源闭环。
- 视觉参考只允许吸收信息架构和交互层次，禁止使用第三方专有名称、贴图、模型、文本或可识别资源。
- 机器外观要逐步替换为真实工业模型和精细贴图，避免紫黑块、黑底透明贴图和文字硬贴图。
- 星球维度资源必须服务阶段进度，禁止在月球、火星、水星等工业星球中乱塞终局矿物。

## 4. 能源与导管系统

### 4.1 SU 机械能

阶段 1-3 的机器按每 tick 实时产出 SU，并立即推送给相邻接收方或 SU 导管。没有接收方时按空转损耗丢弃。高级动能缓存器是当前设计中第一个允许存储 SU 的独立设备。

后续重点：

- 统一 SU 管道、木制旧管道与导管束系统。
- 确保机械齿轮箱能稳定接收多个来源输入。
- TOP 显示实时输入、输出、空转与连接面。
- SU -> FE 转换必须放在合适阶段，避免早期廉价跨能源。

### 4.2 导管束

当前方向是“一个方块格子内可共存多种导管”的复合导管系统。导管不再每阶段廉价铺满，而按材料和科技节点稀疏分布。

计划纳入：

- 物品导管
- 流体导管
- SU 动力导管
- FE 电力导管
- 热力导管
- 气体导管
- 高级能量导管
- 信号/红石导管

后续重点：

- 导管束贴图继续提质，连接面动态出现。
- 扳手可调整输入、输出、关闭、抽取优先级。
- 导管必须接入真实 Capability，至少能和一类真实机器互通。
- Tooltip 全本地化，禁止残留 `%s`、问号或键名。

## 5. 研究、知识库与算力

### 5.1 当前定位

研究系统改名为 UltraTech 自有体系：“量研矩阵”。它吸收复杂研究终端的信息层次，但不使用第三方专有命名、贴图、模型或文本。

核心分工：

- 研究进度保存在 `PlayerResearchSavedData`，绑定玩家。
- 研究终端不再插芯片，只读取玩家知识库和玩家绑定网络算力，产出研究数据。
- 推理计算终端插入算力逻辑处理器和算力储存单元，生成并写入玩家网络算力。
- 算力广播终端用于查看/绑定玩家网络算力。
- 研究木桌负责阶段 1-4 的零散知识碎片与知识书整理。
- 扫描仪终端处理中后期遗迹资料、星际商人资料和星球资料。

### 5.2 量研矩阵 GUI 方向

当前研究终端已经改为更高的独立 GUI，玩家物品栏下移，避免和研究面板重叠。页面结构：

- 概览：网络算力、研究需求、功率消耗、当前队列。
- 科技图谱：左侧节点索引、右侧节点详情、底部资源/权限矩阵提示。
- 机器页：显示附近阶段的机器解锁状态。

后续体验目标：

- 科技树有清晰节点、依赖线、机器解锁、技术解锁和资料来源。
- 每台机器或技术解锁都可以给后续研究提供微量效率加成。
- 前期零散知识、研究木桌、研究终端、扫描仪终端分工清晰。
- 后期资料要求来自其他星球遗迹或星际商人。
- GUI 只通过本地化绘制文本，不把文字烘焙进贴图。

### 5.3 研究节奏

总体流程目标约 1000 小时。研究曲线应让玩家通过机器建设、材料链、星球探索与算力网络逐步加速，而不是只靠原地等进度条。

当前方向：

- 早期阶段 1-4：知识书碎片 + 空白知识书 + 研究木桌解锁零散技术，再通过研究数据学习完整阶段。
- 中期阶段 5+：研究终端消耗玩家网络算力与能量。
- 后期：扫描仪终端、星球遗迹资料、星际商人资料、专用数据源共同参与。
- 研究数据右键学习后写入玩家知识库。
- 分享芯片只能分享比自己低五级的研究等级，制作/编码会降低自身一级。
- 毁灭级研究阶段清空芯片用于高风险重置/测试。

## 6. 物流、储存与流体系统

当前已有或规划：

- 多页物品储存箱，风格参考实用型多页仓储，但要形成 UltraTech 自己的外观语言。
- 各级流体储罐，GUI 向热力式储罐靠近：液位清晰、输入输出面明确。
- 创造无限水、岩浆以及多流体储罐。
- 液体容器和水桶可右键实体方块导入/导出，也可在 GUI 中用光标右键交互。
- 储罐支持输入/输出面调整，扳手能调整导管或储罐面配置。

后续重点：

- 储存箱 Tooltip 页数/槽位格式统一，杜绝 `%s` 残留。
- 储罐顶部、底部、侧面贴图区分。
- 流体、气体、热力、高级能量全部接入真实传输逻辑。
- 所有机器 `quickMoveStack` 必须完整，避免 GUI 交互割裂。

## 7. 材料与制造体系

当前已有大规模材料族：

- 原生金属、合金、矿石宿主、粗矿、粉碎/洗矿/冶炼方向。
- 板、杆、齿轮、螺丝、环、外壳、线缆、丝等基础件。
- 齿轮组作为机械齿轮箱消耗件，具有耐久与最高齿比限制。
- 金属丝系列进入算力逻辑处理器、算力储存单元等核心配方，不只做装饰。

规则：

- 合金不自然生成，必须走机器或合金工艺。
- 终局材料不能出现在月球、火星、水星等工业星球低层矿物中。
- 高级材料必须依赖前级加工链和星球资源，不能靠单步合成廉价获得。

## 8. 航天与星球维度

### 8.1 航天系统

当前航天进入方式：

- 轨道升空器提供平滑上升过程。
- 玩家升至 Y >= 1000 后，在满足导航条件时进入目标维度。
- 航天器导航台校准星图，并消耗玩家网络算力定位目标星球。
- 星图记录目标天体、降落坐标、返航锚点、研究门槛和环境风险。
- 返航必须使用定制序列，不再依赖原版传送动画体验。

### 8.2 月球

目标：

- 真空、无氧、无自然水。
- 低重力。
- 月壤、月海玄武岩、斜长岩、角砾岩、KREEP、水冰、钛铁矿、氦-3。
- 不穿宇航服和氧气罐会窒息。
- 极地永久阴影区和水冰服务制氧链。

### 8.3 火星

目标：

- 稀薄大气、低温尘土、氧化铁地表。
- 需要探测器和航天定位终端定位。
- 后期遗迹资料作为中后期研究来源。

### 8.4 水星

目标：

- 近真空、极端日夜温差、强太阳风。
- 卡洛里盆地、蜘蛛地形、峭壁系统、热力裂缝、永久阴影水冰。
- 内核区域只产出铁、镍、硫化物、钛、铬等普通工业金属，禁止终局矿物。

### 8.5 维度调试

已有方向：

- 创造星球重置印章“星空古路帝印”，用于指定维度重置。
- 航天创造选项卡集中放置航天调试和创造类工具。

## 9. GUI 与视觉规范

必须避免：

- 把文字直接做进 GUI 贴图。
- GUI 物品槽压在进度条、按钮或数据面板里。
- 贴图透明区域显示黑色。
- 语言文件出现 `???`、乱码或 `%s` 残留。
- 模型缺失导致紫黑块。
- 直接使用第三方名称、资源或可识别界面资产。

推荐规范：

- 所有 GUI 文案走本地化键。
- 鼠标悬停显示详细数值。
- 状态灯表达正常、空转、维护不足、缺资源。
- 复杂机器使用侧边栏或分页。
- 重要机器用专用模型和 64x64 以上精细贴图。
- 研究终端使用 UltraTech 自有“量研矩阵”术语和视觉语言。

## 10. 创造与调试内容

当前已有/规划：

- 创造能源核心：超高输出能源，用于测试。
- 创造算力处理器和创造算力储存。
- 创造无限水/岩浆/多流体储罐。
- 创造研究等级提升物品与命令。
- 研究清空命令和各等级研究指令。
- CDK 兑换机制与特殊兑换物品。

这些内容应集中在调试或对应系统创造栏中，不应混入正常生存合成链。

## 11. 近期优先级

### P0：稳定性与闭环

- 清理全部问号、乱码、未本地化 Tooltip。
- 排查紫黑块和透明贴图黑底。
- 跑 DataGen、build、runClient，记录实际问题。
- 检查所有方块/物品是否在合理创造栏中。
- 验证导管能连接至少一类真实机器。

### P1：研究与算力

- 继续精修 UltraTech 量研矩阵、算力网络终端和科技图谱。
- 让研究列表、机器节点、知识库、算力消耗、资料来源形成统一体验。
- 让前期零散知识、研究木桌、研究终端、扫描仪终端清晰分工。
- 避免任何第三方专有命名和资源复用。

### P2：早期机械体验

- 畜力磨盘：动物牵引、拴绳绑定、追踪杆旋转、饲料时间与磨粉反馈。
- 水车：真实木水车贴图、水花与旋转速度。
- 齿轮箱：多源输入、齿轮组耐久、齿比限制、状态灯与 GUI 布局。

### P3：导管与储存

- 导管束模型继续提升，连接面动态出现。
- 扳手统一配置导管/储罐/机器面。
- 热力、气体、高级能量先规划稀疏材料节点，再逐步实现。

### P4：星际探索

- 月球、火星、水星地形继续实测。
- 航天定位终端 GUI 增加行星选择、探测器、算力接入状态。
- 星球遗迹与星际商人资料接入中后期研究。

## 12. 开发验收清单

每次新增内容完成前至少检查：

- 注册 ID 是否稳定。
- 创造栏是否合理。
- `zh_cn.json` / `en_us.json` 是否 UTF-8、无问号、无 `%s` 残留。
- 物品是否有模型、贴图、Tooltip。
- 方块是否有 blockstate、模型、贴图、loot、挖掘 tag。
- GUI 是否无重叠、无文字贴图、无槽位压进度条。
- Capability 是否能真实连接。
- DataGen 是否能运行。
- `gradlew build` 是否通过。
- GUI、维度、渲染、世界生成、网络和 Capability 改动后必须跑 `runClient`。
