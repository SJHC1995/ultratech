# UltraTech 全量配方重构方案

> **状态：分析与方案阶段，尚未批量改写配方。**  
> 本文是 2026-07-23 的可执行设计基线。它把 257 个整合包研究资产转换为 UltraTech 自己的结构规则，不复制外部配方输入、输出、概率、能耗、脚本、任务文字或资源。任何数字化配方改动都必须在本方案逐条确认后落在现有 DataGen 源，而不是手改生成 JSON。

## 证据边界与阅读说明

### 证据来源

1. `docs/ULTRATECH_P0_P4_PRIORITY_SCORECARDS_257_PACKS_2026-07-21.md`：257 个样本的原创评分卡。P0 重点样本为 **GT New Horizons、GTQT 1.8.7、StellarTech 1.0.6.2、NovaEngineering World 1.19.2**；P1-P4 是经过脱敏的结构趋势样本。
2. `docs/ULTRATECH_256_PACK_INDUSTRIAL_CHAIN_REPORT_2026-07-21.md`：256 包的原创工业模式卡，提供 `automation_transition`、`machine_chain`、`multi_step_transformation`、`recovery_outlet`、`flow_and_backpressure`、`energy_infrastructure` 等结构标签。
3. 当前仓库 DataGen 输出：3,571 条运行时配方清单、19 个配方原型、17 条工业路线、20 个工业连接、34 台阶段机、180 条研究。以 Java 17 对当前 `IndustrialProcessMachine.MACHINES` 运行时清单复核，当前工业机实际为 **556 台**，分布于 49 个 `IndustrialProcessModule`；因此“222 台工业机”和此前“346 台”的静态估算都只能视为历史口径，不能再当作当前仓库事实。本文按 **34 台阶段机 + 556 台当前工业机** 制定覆盖方案，并保留 222 作为旧计划的最低覆盖线。

### 不做什么

- 不把 GTNH、E2E、PO3、Nomifactory、DDSS、SevTech 或任何其他包的原始配方照搬进 UltraTech。
- 不以“层数更多”为目标。增加一步只有在它给出批量、并行、热管理、物流、回收、能源或研究能力选择时才成立。
- 不新增平行科技树，不漂移现有 30 阶段、6 里程碑、700 小时和 FloPS/能量分离语义。
- 不把旧存档中的已获得设备变为无效物；配方替换必须有迁移或同阶段回收出口。

### 采用总规则

| 规则 | 对应样本结构 | UltraTech 落地标准 |
|---|---|---|
| 能力先于材料税 | GTNH / GTQT 的持续约束、Nova 的引导图 | 每个阶段门都要有“能否稳定运行”的验证，不只是一组提交材料。 |
| 自动化先于重复 | 257 卡中的 `automation_transition`、`automation_exit` | 玩家在需要高频重复前，必须看到第一台自动化消费者或并行入口。 |
| 共享件先于孤岛件 | `machine_chain`、`distribution_manifest` | 一个中间件优先服务 3–5 个同阶段或相邻阶段消费者；只服务一个终局装置时必须标为专用件。 |
| 替代先于强制单线 | GTQT / Nova 的路线边界信号 | 关键材料或机器至少给出“稳定但慢”与“昂贵但快”两类出口。 |
| 恢复先于惩罚 | GTNH / Stellar 的 `recovery_outlet`、`non_destructive_recovery` | 每个副产物、停机状态、过量中间物都必须能回收、消纳或明确处置。 |

### 未知流程的检索与采纳协议

1. 当某一机器或中间件的真实流程未知时，先查现有 UltraTech 路线、DataGen 和工业模式卡；已有语义能承载时不得新造同义物。
2. 仍未知时，只查两类资料：**样本原作者公开的配置/仓库结构**与**官方机构、国家实验室、国际组织的技术资料**。样本资料只决定玩法结构；真实工业资料只决定流程角色。
3. 每个拟入库配方必须留下 `sample_pattern_source`、`industrial_process_source`、`ultratech_route`、`player_choice`、`recovery_outlet` 五个字段或等价数据。
4. 找不到足够证据的流程不进入主线；最多作为档案、研究说明或后续候选，不能以“看起来合理”直接写成配方。
5. 核、聚变、高压气体、危险化学相关内容永久保持游戏化抽象，绝不在文档、数据包或 UI 中提供可操作现实参数。

### 高保真工业模拟的实现边界

本轮不满足于“物品 A + 物品 B = 机器 C”的贴皮配方。每条工业线要尽可能复现真实流程的**状态变化与运行关系**，但将现实危险参数抽象为游戏等级、工艺状态和设备维护：

| 真实流程维度 | UltraTech 数据/玩法表达 | 不采用的现实细节 |
|---|---|---|
| 物料状态 | 固体、粉体、浆料、液体、气体、蒸汽、冷凝液、熔体、装配件、废料。 | 精确化学配方、浓度和危险物操作法。 |
| 单元操作 | 破碎、筛分、洗涤、烧结、熔炼、铸造、压延、蒸发/冷凝、蒸馏、反应、分离、回收。 | 真实设备尺寸、流体动力学参数。 |
| 连续/批式 | 以机器缓存、管线、批次、预热、稳定运行窗口与输出背压模拟。 | 真实工业控制回路参数。 |
| 热与能 | 环境/预热/高温/低温/深冷等级，以及 FE、SU、蒸汽、热、FloPS 的既有独立账本。 | 温度、压力、功率、临界值等可操作数值。 |
| 可靠性 | 失压、冷凝回流不足、催化剂失活、堵塞、泄漏、失超、满载、维护件磨损。 | 现实事故处置、危险化学或核安全操作程序。 |
| 产线闭环 | 原料 → 主产物 → 副产物/废热/废液/废件 → 回收、再用、低级化或处置。 | 真实排放限值、毒理、处置标准。 |

以下“真实流程骨架”是之后逐机改配方时不可跳过的工艺顺序：

| 工业线 | 必须仿真的流程骨架 | 对应当前路线/机器 |
|---|---|---|

## 一、257 个包配方模式分析报告

### 1.1 嵌套深度分布

#### 样本模式

257 包研究不保存外部配方树，但 P0 与高置信模式卡一致显示出三种可复核的结构区间：

- **短链（3–4 个有效变换）**：用于早期工具、第一台动力机、基础管线和首个可自动化中间品。重点不是“少”，而是让每步都改变玩家能力：手工制备 → 机械制备 → 连续供料。
- **标准工业链（5–7 个有效变换）**：用于金属、热工、化工、电力和精密件。每两到三个变换要出现一次可感知收益，例如批量、稳定供料、能量密度、自动化或副产物回收。
- **长链（8–12+ 个有效变换）**：只用于核循环、低温超导、场约束、宇宙材料和终局预制件。长链成立的条件是：中间物复用、失败可恢复、至少一个副产物流向相邻路线、每段均有能力门，而不是把最终物品拆成十次手搓。

GT New Horizons 与 GTQT 的研究卡强调多约束共存：能源、物流、热/压、批量和维护必须至少有两项在长链中真实生效。StellarTech 的卡提醒，长链一旦没有回收出口，会把“工业深度”退化为纯消耗；NovaEngineering World 的卡则强调链条必须有清晰目标和可见下一步。

#### 当前 UltraTech 基线

对 `src/generated/resources/data/ultratech/recipes` 做了去循环强连通分量压缩后，当前配方图得到：

| 指标 | 当前值 | 解读 |
|---|---:|---|
| 具有 UltraTech 产物的配方节点 | 3,302 | 包含基础件、加工件、机器、研究物和终局件。 |
| 有效配方输出记录 | 3,440 | 同一物品可以拥有多个来源。 |
| 环状分量 | 73 个，最大 4 节点 | 多来自回收/再加工，不应把它误算成无限深度。 |
| 去循环后的平均深度 | 4.22 | 总体落在样本标准工业链下沿。 |
| 中位深度 | 2 | 大量支撑物品过于扁平，容易让阶段机“材料堆叠化”。 |
| 最大深度 | 34 | 仅应出现在终局，需重点检查是否由孤岛件堆叠造成。 |

结论不是“把平均深度拉高”。正确目标是把阶段 1–18 的关键机器构造控制在 **3–7 个有效变换**，把 19–30 的长链限制在可复用的场、线圈、控制、低温、回收模块上。深度为 0–2 的基础支撑件保留，但不应承担过多后期机器的唯一材料税；深度超过 10 的节点必须有共享消费者、替代路径或回收解释。

#### 重构准则

1. 每台阶段机的制造链应当至少含一个“能力件”而非纯金属板：例如密封、控制、换热、绝缘、线圈、轴承、传感或场约束件。
2. 早期不以长链补难度；阶段 1–6 每台关键设备最多新增一类共享中间品。
3. 中期以“两个共享中间件 + 一个专用验证件”取代“八种一次性零件”。
4. 终局允许专用预制件，但必须在同一阶段至少服务两项设施，或产生回收/维护消费者。

### 1.2 材料重叠模式

#### 样本模式

257 包的高置信结构卡反复出现“中间品服务多个机器”的模式：标准紧固件、管线件、线圈/导线、密封件、框架件、控制件、耐热/耐腐蚀件不是某一台机器的私有税，而是形成跨路线的工业语言。GTNH 类样本把共享件作为扩产与替换设备的基础；NovaEngineering World 类样本更强调共享件必须在任务/引导中告诉玩家“下一次还会用到”。

可采纳的经验区间是：**一个常规中间品服务 3–5 个消费者**；超过这个区间时要避免形成单一材料霸权，低于这个区间时要判断它是否应并入通用件、成为副产物消费端，或明确标记为终局专用件。

#### 当前 UltraTech 基线

| 指标 | 当前值 | 结论 |
|---|---:|---|
| 被其他配方消费的 UltraTech 中间物 | 1,143 | 已具有较大工业底盘。 |
| 平均消费者数 | 4.55 | 总体接近 257 包模式的 3–5 区间。 |
| 只有一个消费者的中间物 | 704 | 数量偏高；其中一部分是合理的终局件，其余需要审计。 |
| 高复用件 | 铜线、钢板、钢齿轮、CPU、钛合金板/齿轮、蚀刻液等 | 应保留，但需要二级替代和吞吐缓冲，避免单点材料霸权。 |

“孤岛中间品”不能一律删除。以下三类应保留为单消费者：最终设备本体、一次性终局钥匙件、明确只服务一个里程碑验收的安全件。以下三类必须进入重构队列：  

1. 深度高、仅服务单个普通机器、没有回收说明的材料；  
2. 同阶段存在语义接近的多个专用件，却没有共享消费者的材料；  
3. 被作为阶段门材料税使用，但没有改变设备能力的材料。

优先审计队列包括 `precision_ceramic_part`、`quantum_logic_controller`、`tokamak_ring`、`laser_fusion_chamber`、`proton_accelerator` 等高深度节点；不是要把这些终局/巨构设备强行通用化，而是要检查它们前面的陶瓷、控制、线圈、密封、低温和场约束件能否服务相邻设施。

### 1.3 替代路线比例

#### 样本模式

P0 卡中 `automation_exit`、`recovery_outlet`、`flow_and_backpressure` 的共同含义是：玩家被卡住时应该有不同代价的选择，而不是只能回去重复同一条材料链。适合 UltraTech 的替代形式不是复制外部配方，而是复用现有路线契约：

- **稳定但慢**：原料广、吞吐低、能耗/人工维护较低，适合保底、启动和断供恢复。
- **昂贵但快**：高能耗、高纯度材料、更多控制件或并行设备，适合规模化。
- **回收但低效**：把尾料、废液、废热、废催化剂转回低级原料；不能替代主线，但能防止硬锁。

#### 当前 UltraTech 基线

当前生成配方中有 99 个产物拥有两个或更多明确来源，占 3,302 个产物约 **3.00%**。这不能直接与“30–40% 的机器具备替代路线”一一比较，因为大量物品本身不是机器，但仍说明**显式替代配方覆盖偏低**。现有多来源主要集中在钛、镓、铝等金属，生物质/污泥等回收物，及少数报告类产物；关键阶段机与大量工业机仍以单一构造链为主。

本轮目标不要求所有物品双配方，而是要求：

| 对象 | 当前问题 | 目标 |
|---|---|---|
| 34 台阶段机 | 主线构造链多为单线 | 至少 12 台拥有可验证的第二构造/启动路径。 |
| 556 台当前工业机（旧目标口径为 222） | 大量设备为一次性制造税 | 第一轮至少覆盖阶段 1–12 的生产、回收与基础设施机；最终覆盖 556 台中的全部关键生产/回收/基础设施机。 |
| 关键中间物 | 少数金属/导线承担过多压力 | 每个高复用节点增加一个低速保底或高投入快线。 |
| 副产物 | 可能只在个别工艺中可见 | 每一类副产物至少有一个回收、发电、建材化或无害化出口。 |

### 1.4 阶段门类型分布

#### 样本模式

257 包的结构证据可归纳为两种门：

1. **材料税型门**：要求稀有材料或大量重复件，但不证明玩家已经能稳定运行对应能力。它可以作为少量成本，但不能独立承担阶段门。
2. **能力验证型门**：要求累计产出、稳定运行、热/压控制、输入输出闭环、回收处理、研究/算力贡献、并行吞吐或结构完整。它把“拥有材料”与“掌握工业能力”区分开。

GTNH、GTQT 与 NovaEngineering World 的高优先级结构都支持第二种。StellarTech 的恢复不足也提示：若门只验证提交，不验证恢复，玩家一旦断料或停机就会失去对系统的理解。

#### 当前 UltraTech 基线与缺口

当前 `ProgressionAudit` 能验证阶段预算、配方存在性、消费者、回收、路线契约、里程碑等结构风险，结果为 0 项结构风险；但配方元数据尚未显式记录每一个阶段门是“材料税”还是“能力验证”。因此不能诚实地声称当前二者的精确比例。

本方案要求把阶段门数据化为：

```text
gate_type = MATERIAL_TAX | CAPABILITY_VALIDATION | HYBRID
validation_metric = cumulative_output | uptime | recovery_cycle | parallel_throughput |
                    thermal_stability | pressure_stability | compute_contribution |
                    field_containment | multiblock_integrity
```

目标分布为：**能力验证型不少于 40%，混合型不少于 35%，纯材料税型不高于 25%**。纯材料税只能承担视觉/叙事成本，不能成为主线唯一锁。

## 二、阶段 1–6 重构方案：基础动力与蒸汽启动

### 2.1 样本模式提取

来源：GT New Horizons 的机械准备/自动化转折结构、NovaEngineering World 的显式目标图、GTQT 的物流边界结构、256 卡的 `machine_chain` 与 `automation_transition` 标签。采纳的是“手工准备 → 可持续动力 → 流体/压力闭环”的顺序，不采纳外部具体配方。

阶段 1–6 应形成四种共享能力件：**传动、容器/密封、基础换热、压力安全**。它们要被畜力磨盘、水车、风车、齿轮箱、低压锅炉、高压锅炉共同使用，而不是为每台设备创造一套私有部件。

### 2.2 当前差距分析

当前阶段 1–6 的构造已经有 `wooden_frame`、`wooden_gear`、`iron_bolt`、`copper_pipe`、`safety_valve`、`pressure_gauge`、`steel_pressure_vessel` 等良好基础；但阶段 1–4 的能力验证偏弱，阶段 5–6 的锅炉构造需要更清晰地表现“水质、密封、换热、压力”的递进，而不是仅替换更高级金属。

### 2.3 新增或提升为共享件的清单

| 设计对象 | 来源模式 | UltraTech 原创用途 | 首批消费者 |
|---|---|---|---|
| 机械传动组件组（轴、离合、基础轴承的统一组件） | GTNH `machine_chain`；Nova `guided_objective_graph` | 把畜力、水力、风力、齿轮箱连接成可解释的传动契约。 | 畜力磨盘、水车、风车、齿轮箱、早期压榨/研磨机。 |
| 低压密封包（基于现有橡胶密封/松香/基础纤维路线） | GTQT `logistics_boundary` | 作为流体容器、低压锅炉、阀门与早期水处理的共享件。 | 木罐、陶罐、低压锅炉、基础管线。 |
| 锅炉换热芯 | 257 卡 `energy_bound_processing` | 把“能烧”升级为“能稳定换热”，不直接复制外部换热配方。 | 低压锅炉、高压锅炉、蒸汽涡轮的维护/升级。 |
| 压力安全组件组 | GTNH `visible_operating_constraint` | 由现有安全阀、压力表、容器组合成能力验证件。 | 高压锅炉、蒸汽支路、压缩空气前置设备。 |

### 2.4 配方修改方案

| 阶段机 | 改动方向 | 稳定但慢 | 昂贵但快 | 能力验证 |
|---|---|---|---|---|
| 畜力磨盘 | 保留石磨盘与木框，加入传动组件语义 | 人工/畜力低吞吐研磨 | 多畜力并联或更好轴承提高连续运行 | 连续完成基础粉碎并保持动力。 |
| 水车 | 把轮叶、转子、木框连接为可维护水力组件 | 自然水流，小缓存 | 处理水/导流与更高效叶片 | 稳定 SU 输出与下游磨盘运行。 |
| 风车塔 | 明确风况、传动和轴承，而非只提高材料等级 | 风力间歇，低维护 | 增加调速/储能/更高塔架 | 连续运行窗口与传动比验证。 |
| 齿轮箱 | 从单一齿轮升级为“分配/离合/维护”节点 | 低比率、低损耗 | 高比率、需要更好轴承和润滑 | 同时驱动两个不同负载。 |
| 低压锅炉 | 强化水质、密封、换热三件组 | 普通燃料与处理水，低压蒸汽 | 预热、换热芯、并联炉膛，提高供汽 | 产出蒸汽并回收冷凝水。 |
| 高压锅炉 | 强化压力安全与容器完整性 | 高压但低吞吐保守运行 | 更好容器、仪表、预热和并联供料 | 压力稳定、阀门动作、无泄压堵塞。 |

### 2.5 阶段门建议

- 阶段 1–2：验证“持续 SU 供给”，不能只提交木齿轮。
- 阶段 3–4：验证“单源动力分配到两个消费者”，证明玩家已拥有传动能力。
- 阶段 5：验证“蒸汽产生—使用—冷凝回收”闭环。
- 阶段 6：验证“高压稳定运行 + 安全泄压/停机恢复”。

### 2.6 替代路线建议

1. **水力保底线**：低成本、吞吐受地形约束；用于启动与断燃料恢复。
2. **生物燃料蒸汽线**：消耗更高，但可稳定扩产；接入既有 `biobased_industry`。
3. **尾料/木炭辅助线**：低效率消纳副产物，不能超过主燃料效率。

## 三、阶段 7–12 重构方案：热机、内燃与电气化

### 3.1 样本模式提取

来源：GTQT 的连续热工与物流边界、GTNH 的机器链和自动化出口、StellarTech 的能源基础设施压力。此段的核心不是“发动机越做越贵”，而是让缸套、转子、线圈、换热、润滑和燃料分配成为跨设备复用的工业语言。

### 3.2 当前差距分析

阶段机顺序——蒸汽涡轮、斯特林、活塞内燃、涡轮内燃、燃气轮机、热电、直流换流站——具备正确的能源叙事，但构造层需要更明确的共享热机件与两条燃料/热源路径。否则玩家会感到每台机器只是上一台机器加更多板、齿轮与研究数据。

### 3.3 中间品插入方案

| 中间件组 | 来源模式 | 原创适配 | 消费者 |
|---|---|---|---|
| 热机转子组 | GTNH `machine_chain` | 以轴、叶片/转子、动平衡报告构成，而不是单独的“更高级齿轮”。 | 蒸汽涡轮、燃气轮机、热电辅助、后续压缩机。 |
| 缸套—活塞组 | GTQT `multi_step_transformation` | 使用铸造、加工、润滑三种能力，不使用外部数值。 | 活塞内燃机、涡轮内燃机、泵和压缩设备。 |
| 换热与绝缘组 | Stellar `energy_infrastructure` | 以现有绝缘纸、管路、金属件形成热端/冷端模块。 | 斯特林、热电、锅炉回热、冷却支路。 |
| 燃料调质组 | 256 卡 `recovery_outlet` | 让生物燃料、裂化气、尾气各有稳定/快速/回收入口。 | 内燃机、燃气轮机、化工路线。 |
| 早期电气组件组 | Nova `guided_objective_graph` | 线圈、绝缘、整流/保护件服务热电和直流站。 | 热电、直流换流站、早期电网设备。 |

### 3.4 配方修改方案

| 阶段 | 设备 | 改动 | 稳定但慢 | 昂贵但快 |
|---:|---|---|---|---|
| 7 | 蒸汽涡轮 | 引入热机转子与冷凝回水接口 | 低压供汽、低转速 | 高压供汽、动平衡转子、回热支路 |
| 8 | 斯特林 | 改为热端/冷端/密封三要素 | 固体燃料或余热，低功率 | 高温热源、换热芯、稳定冷端 |
| 9 | 活塞内燃机 | 引入缸套—活塞—润滑闭环 | 生物燃料，低速稳定 | 调质燃料、精制润滑、高功率 |
| 10 | 涡轮内燃机 | 以压缩空气/燃料喷射为能力点 | 低压空气、保守负载 | 高压空气、燃料调质、并行供给 |
| 11 | 燃气轮机 | 加入转子、热端、尾气去向 | 裂化气/低热值气体 | 高纯燃气、回热、尾气再利用 |
| 12 | 热电与直流站 | 以温差稳定与绝缘/整流保护为核心 | 低效率余热回收 | 高温差、稳定散热、快速电气化 |

### 3.5 阶段门建议

- 阶段 7–8：累计产出一定量机械功或蒸汽功，验证回水/冷端没有堵塞。
- 阶段 9–11：连续运行一个燃料周期，验证燃料、空气、润滑、尾气至少三项中的两项。
- 阶段 12：验证 SU/热能到 FE 的转换稳定性与过载保护，而不是仅提交换流站。

### 3.6 替代路线建议

1. 生物燃料—内燃保底线：慢、可再生、适合断油恢复。  
2. 裂化气—燃气轮机快线：高投入、吞吐高、必须有尾气处理。  
3. 余热—热电回收线：效率最低但可把工业余热变成早期 FE 缓冲。

## 四、阶段 13–18 重构方案：电力、研究主线与核裂变

### 4.1 样本模式提取

来源：GTNH 的标准件复用与连续供电、GTQT 的高纯材料/物流门、Nova 的研究目标图、257 卡的 `energy_infrastructure`、`machine_chain` 与 `recovery_outlet`。电力时代不应只有“更多 FE”，而要让绝缘、硅、控制、容器、燃料和回收组成一个可诊断的电力系统。

### 4.2 当前差距分析

现有路线已覆盖 `standard_parts_backbone`、`precision_campus`、`petrochemical_platform`、`electrochemical_polymer` 与阶段 16–17 的魔法主线整合。需要补强的不是另起炉灶，而是让电网/控制件同时服务核前置、精密制造和研究终端，降低单消费者控制件比例。

### 4.3 新增或重组中间品

| 中间件 | 证据来源 | 作用 | 复用目标 |
|---|---|---|---|
| 绝缘—线圈组件组 | GTQT 高纯/电力结构；Nova 引导结构 | 将绝缘纸、线材、线圈、外壳整理为分级组件。 | 光伏、换流站、传感、控制、核仪表。 |
| 硅片—控制板组件组 | 257 `multi_step_transformation` | 把硅、蚀刻、封装、控制从纯材料税变为精度能力。 | 光伏、研究终端模块、核控制、精密机床。 |
| 核容器—控制组 | GTNH 持续约束；Stellar 基础设施压力 | 压力容器、控制棒、屏蔽、仪表组成可检测节点。 | 裂变堆、增殖堆、熔盐前置、废料处理。 |
| 燃料循环接口 | 257 `recovery_outlet` | 把新燃料、乏燃料、回收燃料与玻璃化/封存接起来。 | 裂变、增殖、后续熔盐/聚变材料链。 |

### 4.4 电力与核燃料链重构

**标准燃料路线（稳定但慢）**  
原矿/浓缩 → 燃料制备 → 标准燃料组件 → 受控裂变 → 乏燃料分离 → 低效回收或封存。它的职责是让玩家在不追求极端燃料闭环时仍能稳定进入阶段 18 以后。

**增殖/回收路线（昂贵但快）**  
更高纯材料、额外控制/屏蔽组件、更多研究/物流要求 → 增殖或高利用率燃料组件 → 更高吞吐 → 乏燃料再处理 → 可再次进入燃料链的次级材料。它必须附带泄漏/热/废物缓冲的恢复出口，不能把“高效”做成无风险免费循环。

### 4.5 阶段门建议

- 电力门：验证负载下电网稳定、绝缘/保护件存在、研究终端有实际可用 FE 与 FloPS 供给。
- 光伏门：验证日照或等价环境条件下累计发电，而不是只提交光伏矩阵。
- 核门：验证燃料、控制、冷却、废物四项中至少三项同时闭环；停机后可说明原因并恢复。

### 4.6 替代路线建议

1. 光伏/热电的低密度保底电力；  
2. 标准裂变的稳定基荷；  
3. 增殖/回收的高吞吐路线；  
4. 废热/副产气回收为低效辅助电力，防止单一燃料硬锁。

## 五、阶段 19–24 重构方案：聚变、粒子与极端场

### 5.1 样本模式提取

来源：GT New Horizons 的多约束长链、GTQT 的低温/终局压力、StellarTech 的基础设施与恢复对照、257 卡的 `flow_and_backpressure`、`energy_bound_processing`、`space_or_endgame_gate`。此段必须把难度放在“稳定运行条件”而不是单纯巨大数字。

### 5.2 当前差距分析

现有阶段机包含熔盐堆、氘聚变、氦聚变、质子加速器、托卡马克、激光聚变室。当前风险是这些高深度节点变成彼此仅单次消费的孤岛。应将超导线圈、低温接口、场控制、靶材/窗口、诊断和废热回收提升成跨设施通用能力件。

| 低温服务与回收组 | Stellar `recovery_outlet` 对照 | 液氦/低温介质、回收缓冲、泄压/失超恢复。 | 所有超导/聚变设备。 |

| 激光聚变室 | 低频脉冲、可替换靶材 | 高频靶材供给、高能光学/电容、严格同步 | 靶材残留、光学损耗件、热/辐照维护。 |

- 阶段 21–22：验证可持续粒子/聚变输出和诊断数据贡献。
- 阶段 23–24：验证稳定与爆发路线至少各可选一种，且事故/失超/过热均有明确恢复动作。

### 5.6 替代路线建议

1. 熔盐/裂变余热支撑的慢速低温线；  
2. 高能聚变主线；  
3. 粒子装置提供高投入的快速稀有材料/研究贡献；  
4. 退役线圈、靶材与低温介质的回收线，产率低于主线但避免停机死锁。

## 六、阶段 25–30 重构方案：终局、宇宙材料与巨构

### 6.1 样本模式提取

来源：GTNH、GTQT 的终局长链/多约束模式，StellarTech 的恢复反例，NovaEngineering World 的目标引导，以及 257 卡 `space_or_endgame_gate`、`distribution_manifest`、`non_destructive_recovery`。终局不是“再加十层材料”，而是把此前所有能力——并行、低温、场、算力、物流、回收——重新组合成可维护的巨构系统。

### 6.4 终局目标选择方案

终局必须提供多种“毕业证明”，但它们共用现有 30 阶段和 Genesis Core，不新增第 31 阶段：

| 目标类型 | 现有承载 | 验证重点 |
|---|---|---|

### 6.5 阶段门与替代建议

- 终局门不以“提交宇宙材料”单独成立；必须组合为结构完整、持续运行、回收出口和研究/算力贡献。
- 稳定线：低吞吐、可连续维护、消耗广泛材料。  
- 快线：高能、高算力、高纯材料和更高维护负担。  
- 回收线：把失稳碎片、维护废件、低级宇宙残渣转回预制/密封/缓冲材料，产率永远低于主线。

| `NUCLEAR_FUEL_CYCLE` | 18 | 16–20 | 矿石、转化、分离、芯块、组件、乏燃料、再处理与封存。 |
| `FUSION_FUEL` | 6 | 19–24 | 燃料、靶材、包层和增殖物；稳定线与爆发线共用低温/场控。 |
| `HELIUM3_EXTRACTION` | 6 | 19–24 | 月壤前处理、脱气、分离、储运与尾矿回收。 |
| `SUPERALLOYS` | 8 | 19–24 | 配料、真空/重熔、铸锻、热处理、涂层与报废件回炉。 |
| `BIOCHEMISTRY` | 10 | 13–18 | 预处理、糖化、培养、发酵、分离、纯化和培养基/菌体回收。 |
| `INTEGRATION_FABRIC` | 10 | 16–30 | 魔法工业接口与终局预制；缺模只转纯理论，不能破坏主线毕业。 |

**逐机落地规则。** 上表的 556 台并非“只需模块级口号”。每台机器的构造与加工配方在改动前必须填写 7.11 的证据表；模块契约只决定默认共享件、慢线、快线、回收和验证类型。名称包含 `crusher`、`mill`、`screen`、`filter`、`separator` 的机器优先承接原料预处理和尾料回收；`reactor`、`tank`、`tower`、`cell`、`furnace` 必须显式声明介质/热/压/催化或电化学状态；`press`、`caster`、`molder`、`assembly`、`machining` 必须至少连接一个上游半成品和一个下游真实消费者；`monitor`、`tester`、`evaluation`、`quality` 设备必须成为能力验证或快线门槛，而不是孤岛装饰。

### 7.10 六段工艺状态机与现有材料落点

这一节把“真实工业流程”进一步压缩成可以进入 Forge 数据与 BlockEntity 状态的游戏状态机。每一步都只描述游戏语义，不写现实工艺参数；但机器不能再跳过原料准备、稳定运行和回收这三类真实工业关系。

#### 阶段 1–6：机械—蒸汽状态机

```text
原木/石材/矿粉
  → 木框、木齿轮、石磨盘、螺栓
  → 畜力/水力/风力传动
  → 齿轮箱分配
  → 处理给水 + 燃料 + 低压密封 + 换热
  → 低压蒸汽
  → 高压容器 + 仪表 + 安全阀
  → 高压蒸汽 / 冷凝液 / 可回收热
```

| 现有材料或设备 | 必须承担的真实流程角色 | 样本模式来源 | 重构要求 |
|---|---|---|---|
| `wooden_frame`、`wooden_gear`、`iron_bolt` | 设备骨架、动力传递、可替换紧固件 | GTNH 机械准备 | 不再只是每台机器重复税；至少跨畜力、水力、风力、齿轮箱复用。 |
| `stone_millstone`、畜力磨盘 | 初级粒度减小 | 257 `automation_transition` | 手工粉碎只作启动，畜力要成为第一条连续研磨出口。 |
| 木轮叶、木转子、水车、风车 | 原动机与机械输入波动 | Nova `guided_objective_graph` | UI/提示显示水流、风况、传动负载或缓存，而非单纯“能/不能运行”。 |
| `copper_pipe`、`safety_valve`、`pressure_gauge` | 给水、蒸汽管网、保护与测量 | GTQT `visible_operating_constraint` | 锅炉快线必须额外消耗其中至少两类，并在停机时返回明确原因。 |
| `copper_boiler_shell`、`steel_pressure_vessel`、`steam_piston`、`reinforced_piston` | 容器、换热、做功、耐压升级 | 真实蒸汽系统骨架 + GTNH 能源链 | 低压到高压的区别必须体现在压力安全、预热/回水、吞吐和维护，而不是只改金属名称。 |

**能力验收。** 阶段 1–4 用“累计带动两个不同工序”验证传动；阶段 5 用“蒸汽产出与冷凝回收”验证热工；阶段 6 用“高压稳定窗口 + 受控停机/恢复”验证压力系统。玩家只拥有材料但没有构建运行闭环时，不应获得下一阶段主线。

#### 阶段 7–12：热机—燃料—电力状态机

```text
蒸汽/燃料/余热
  → 热端、冷端、换热器、密封
  → 转子或缸套—活塞组件
  → 机械做功 / 压缩空气
  → 尾气、废热、冷凝水或润滑废料
  → 热电转换 / 整流保护
  → 可用 FE 与电网负载
```

| 阶段机 | 原料/状态重点 | 样本模式来源 | 不可省略的真实流程模拟 |
|---|---|---|---|
| `steam_turbine` | 蒸汽品质、转子、冷凝 | GTNH `energy_bound_processing` | 蒸汽不足、冷凝背压、转子维护三种状态不能合并成“无能量”。 |
| `stirling_engine` | 热端、冷端、密封 | Stellar `energy_infrastructure` | 温差是能力；余热可运行但吞吐低，高温稳定热源可快速运行。 |
| `piston_combustion_engine` | 缸套、活塞、润滑、燃料 | GTQT `multi_step_transformation` | 燃料、空气、润滑至少两项影响稳定性；不可只喂一个燃料物品。 |
| `turbo_combustion_engine` | 压缩空气、燃料喷射、尾气 | 257 `flow_and_backpressure` | 压缩空气不足应降低吞吐；尾气要进入回热、净化或低效发电。 |
| `gas_turbine` | 燃气品质、转子、热回收 | 真实 CHP 骨架 + GTNH | 快线不只是更快燃料，必须有回热/余热利用或更高维护成本。 |
| `thermoelectric_generator` | 温差、导热、绝缘、散热 | 257 能源基础设施 | 只能回收已有热差；不能作为无燃料的万能发电机。 |
| `dc_converter_station` | 整流、绝缘、保护、负载 | Nova 电力目标 | 需要实际 SU/热机输入、FE 输出和过载/保护反馈。 |

**两路约束。** 生物燃料/余热路线是稳定但慢的恢复线；调质燃料、压缩空气、回热与更高等级转子构成昂贵但快的扩产线。两线的共享中间件应是密封、润滑、绝缘、管路和标准件，而不是彼此完全隔离的“燃料专用零件”。

#### 阶段 13–18：电网—光伏—裂变状态机

```text
导线/绝缘/硅基板/控制件
  → 发电或变换设备
  → 负载、研究终端、工业网络
  → 燃料制备与组件
  → 受控裂变
  → 乏燃料贮存
  → 回收再利用 / 固化处置
```

| 现有对象 | 真实流程映射 | 样本模式来源 | 重构限制 |
|---|---|---|---|
| `photovoltaic_matrix` | 基板/单元/封装/互连/并网 | NREL 光伏制造骨架 + Nova | 光伏矩阵必须有硅、封装和电网接口；日照/环境与维护决定保底吞吐。 |
| `fission_reactor` | 标准燃料、控制、冷却、乏燃料 | IAEA 燃料循环抽象 + GTNH | 不模拟现实核细节；只以燃料组件、控制、冷却、贮存和处置作为游戏状态。 |
| `breeder_reactor` | 高投入燃料利用/回收循环 | GTQT 回收与批量模式 | 高吞吐路线增加控制、屏蔽、冷却、再处理和废物账本；不可形成无代价复制。 |
| 研究终端与主线魔法节点 | 电网/算力与跨域工业接口 | Nova 引导 + 现有魔法主线 | 魔法节点是主线必修，缺模时为纯理论而非阶段死锁。 |

**能力验收。** 阶段 13–15 验证电网在负载下工作而非只合成方块；阶段 16–18 验证燃料—运行—乏燃料/回收或处置。每个核相关快线要有“安全停机后能重新启动”的恢复出口。

#### 阶段 19–24：低温—场—聚变/粒子状态机

```text
高阶电力 + FloPS
  → 低温服务
  → 超导线圈 / 场控制 / 容器
  → 燃料或靶材准备
  → 稳定连续运行 或 脉冲爆发运行
  → 诊断数据、主产物、热负荷、耗材残留
  → 低温介质回收、线圈/靶材回收、维护重启
```

| 阶段机 | 稳定但慢的模拟 | 昂贵但快的模拟 | 回收/故障状态 |
|---|---|---|---|
| `molten_salt_reactor` | 低流量、低温度等级、较低产出 | 净化、耐腐蚀件、泵阀并行 | 熔盐污染、泵阀维护、容器泄漏恢复。 |
| `deuterium_fusion_reactor` | 连续小批、低场 | 更高低温/线圈/算力负载 | 低温服务不足、场不稳、燃料中断。 |
| `helium_fusion_reactor` | 稳定研究级运行 | 高投入脉冲与更高维护 | 靶材/燃料残留、热负荷、场恢复。 |
| `proton_accelerator` | 低能束流提供研究/前处理 | 高能束流提供快速高阶产物 | 束流损失、屏蔽与冷却维护。 |
| `tokamak_ring` | 持续低输出供给 | 高场高输出并行线圈 | 失超碎片、冷却剂、热负荷回收。 |
| `laser_fusion_chamber` | 低频脉冲、少量靶材 | 高频靶材、光学/控制阵列 | 光学耗材、靶材残渣、充能失败恢复。 |

#### 阶段 25–30：巨构—终局循环状态机

```text
高阶材料/预制件
  → 巨构框架、端口、控制、场稳定
  → 启动算力与连续能源
  → 宇宙/虚无/奇异材料生产
  → 维护废件、失稳碎片、热/场残留
  → 低级回收或维护预制件
  → 下一轮巨构或 Genesis 验收
```

| 阶段机 | 必须共享的能力 | 样本模式来源 | 终局不退化为材料税的条件 |
|---|---|---|---|
| `strange_matter_compressor`、`degenerate_matter_generator` | 场稳定、算力、巨构预制、维护回收 | GTNH/GTQT 终局长链 | 快线必须提高并行与维护负荷，而非单纯增加输入数量。 |
| `antimatter_collector`、`casimir_extractor` | 容器/真空或抽象场、低温、屏蔽/控制 | 257 极端约束 | 故障产生低级回收物；不能无提示清空全部进度。 |

| `cosmic_gate`、`nihility_engine`、`genesis_core` | 巨构端口、同步控制、持续算力、终局循环 | Nova 终局目标图 + 现有研究树 | Genesis 保持阶段 30 双必修；其他目标作为毕业形态而不是第 31 阶段。 |

### 7.11 逐批落地时的配方证据表模板

每一条真正写入 DataGen 的配方、替代配方或处理配方都必须先填下表；没有填表的改动不得以“真实工业感”为理由合入：

| 字段 | 必填内容 |
|---|---|
| `recipe_id` | UltraTech 自己的稳定 ID。 |
| `target_machine` | 34 台阶段机之一，或 556 台工业机之一。 |
| `sample_pattern_source` | 如 `MP-GTNH-01 / machine_chain + automation_exit`，只记录结构模式。 |
| `industrial_process_source` | 如 `IR-STEAM-01 / boiler-condensate-return`，只记录流程角色。 |
| `existing_route` | 必须是既有 17 条 `IndustrialRoute` 之一。 |
| `shared_intermediate` | 复用的现有材料；若新增，列出至少两个消费者。 |
| `slow_route` | 稳定但慢：原料、运行约束、产物、恢复出口。 |
| `fast_route` | 昂贵但快：增加的真实能力成本，不得只是数量膨胀。 |
| `gate_type` | `MATERIAL_TAX`、`CAPABILITY_VALIDATION` 或 `HYBRID`。 |
| `validation_metric` | 产出累计、运行时间、稳定窗口、回收循环、并行吞吐、结构完整性等。 |
| `failure_and_recovery` | 缺料、缺能、缺 FloPS、堵塞、热/压/场不稳、输出满载对应的恢复动作。 |

## 八、确认前检查清单

- [ ] 每一项新增中间件都能映射到至少一个 257 包结构模式和一个现有 `IndustrialRoute`。
- [ ] 每一项新中间件均已核对现有 `IndustrialProcessMaterial`，不存在同义重复。
- [ ] 每台被重构的阶段机都保留旧存档恢复或同阶段回收出口。
- [ ] 每项替代路线都明确“稳定但慢”与“昂贵但快”的成本差异，且快线不是免费升级。
- [ ] 阶段门有 `validation_metric`，不再只依赖材料提交。
- [ ] 34 台阶段机与 556 台当前工业机的逐机改动表在实施前补齐；旧 222 台和此前 346 台口径只作为历史覆盖下限，并以 DataGen 源为唯一修改入口。
